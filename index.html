Introduction

This project calculates RFM (Recency, Frequency, Monetary) values for customers and uses these insights to recommend personalized plans or offers. The system is built using:

  • Django for the backend API.
  • React for the frontend user interface.
  • Firebase for storing and managing customer data.

Features

  • Calculates RFM values for customers based on their activity and spending.
  • Stores and manages customer data in Firebase (Firestore).
  • Recommends personalized plans and offers based on RFM segmentation.
  • Provides a user-friendly interface using React.
  • Flexible and scalable backend using Django API.

Setup

Before you start, you will need the following tools and software:

  • Python 3.x for the Django backend.
  • Node.js for the React frontend.
  • Firebase account to store customer data and interact with Firestore.

Installation

Follow these steps to set up the project on your local machine:

  1. Clone the repository from GitHub:
    2. git clone https://github.com/yourusername/rfm-calculation-project.git
  2. Install Django and required Python dependencies:
    3. cd rfm-calculation-project/backend
pip install -r requirements.txt
  3. Set up Firebase and configure it with your Django backend (refer to Firebase documentation for details on setting up Firestore).
  4. Install frontend dependencies using npm:
    5. cd ../frontend
npm install
  5. Start the backend Django server:
    6. python manage.py runserver
  6. Start the React development server:
    7. npm start

Usage

Once the installation is complete, you can start using the system:

  • Enter customer information such as Name, Phone Number, Cost, and the Date of the last purchase.
  • The system will calculate the RFM values for the customer based on the provided data.
  • Based on the RFM values, personalized plans or offers will be suggested to the customer.
  • View RFM data on the frontend dashboard to analyze customer segmentation.
